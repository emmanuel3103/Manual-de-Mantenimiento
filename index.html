Resumen

Este documento constituye el Manual de Mantenimiento del sistema EDI-301, incluyendo procedimientos preventivos y correctivos, plan de actualizaciones, manejo de errores y estrategias de respaldo. Su objetivo es asegurar la continuidad operativa y evolución estable del sistema.

Propósito

El propósito de este manual es establecer las directrices formales para el mantenimiento del sistema EDI-301, asegurando que las tareas preventivas, correctivas, perfectivas y adaptativas se ejecuten de forma ordenada, eficiente y documentada.

Alcance

Este manual abarca actividades de mantenimiento, registro de errores, actualizaciones, respaldos, recuperación ante fallos y procedimientos técnicos. Está dirigido al personal técnico responsable del sistema EDI-301.

1. Procedimientos de mantenimiento

1.1 Tipos de mantenimiento

  • Correctivo: solución de errores detectados.
  • Preventivo: revisiones periódicas.
  • Perfectivo: mejoras funcionales y optimización.
  • Adaptativo: ajustes ante nuevas tecnologías.

1.2 Procedimiento general

  1. Registrar incidente en GitHub.
  2. Clasificar prioridad y tipo.
  3. Planificar cambios y pruebas previas.
  4. Implementar corrección/mejora.
  5. Ejecutar pruebas unitarias e integradas.
  6. Desplegar con control de versiones.
  7. Documentar y actualizar bitácoras.

1.3 Frecuencia de revisión recomendada

  • Logs: semanal.
  • Dependencias: trimestral.
  • Seguridad y roles: semestral.
  • Respaldo-recuperación: semanal.

2. Plan de actualizaciones

2.1 Objetivo

Garantizar la evolución continua del sistema manteniendo compatibilidad, seguridad y rendimiento óptimo.

2.2 Estrategia general

  • Uso de control de versiones Git.
  • Ramas main, dev y feature.
  • Registro de cambios en CHANGELOG.md.
  • Pruebas automatizadas previas al despliegue.
  • Notificación de cambios relevantes.

2.3 Cronograma sugerido

Periodo Actividad principal Responsable
Mensual Revisión de dependencias y librerías Equipo técnico
Trimestral Actualización de API REST y módulos Flutter Desarrolladores backend / frontend
Semestral Evaluación de rendimiento y auditoría de seguridad Administrador del sistema
Anual Revisión integral de arquitectura y documentación Coordinador del proyecto

3. Manejo de errores

3.1 Registro y monitoreo

  • Middleware para capturar excepciones.
  • Logs locales o servicios externos.
  • Alertas de fallos críticos.

3.2 Estructura del mensaje de error

{
  "ok": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "El campo 'email' es obligatorio",
    "details": ["email"]
  },
  "timestamp": "2025-10-28T12:00:00Z"
}

3.3 Tipos de error comunes

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found
  • 409 Conflict
  • 500 Internal Server Error

3.4 Procedimiento ante errores críticos

  1. Identificar módulo y reproducir error.
  2. Revisar logs y versión desplegada.
  3. Aplicar hotfix temporal.
  4. Implementar solución definitiva.
  5. Notificar usuarios/responsables.

4. Copias de seguridad y recuperación

4.1 Estrategia de respaldo

  • Backups diarios de base de datos.
  • Respaldo semanal de archivos.
  • Código sincronizado con Git.
  • Copias externas en nube.

4.2 Plan de recuperación

  1. Detener servicio afectado.
  2. Restaurar copia verificada.
  3. Validar integridad.
  4. Registrar evento.
  5. Notificar restablecimiento.

4.3 Pruebas de recuperación

Realizar simulaciones de restauración al menos cada semestre.

Conclusión

Este manual define procesos fundamentales para el mantenimiento del sistema EDI 301, asegurando continuidad operativa, seguridad y mejora constante dentro del entorno institucional.
